## 实现付费下载配置功能

### 1. 修改配置系统
- **添加PaymentConfig结构体**：在`config.go`中添加支付配置结构体，用于管理不同服务的收费模式
- **支持按次收费配置**：允许配置不同服务的单次收费金额
- **支持免费模式**：允许配置特定服务是否免费

### 2. 环境变量设计
```env
# 支付配置
# 格式：SERVICE_NAME=PRICE_PER_USE
# 0表示免费，>0表示按次收费
PAYMENT_PAPER_DOWNLOAD=0          # 论文下载（0免费，x表示x元/次）
PAYMENT_FORMAT_CHECK=10           # 格式检查（10元/次）
PAYMENT_FORMAT_FIX=15             # 格式修复（15元/次）
PAYMENT_REPORT_DOWNLOAD=5         # 报告下载（5元/次）
```

### 3. 代码修改
- **配置加载**：在`config.go`的`LoadConfig`函数中添加支付配置的加载逻辑
- **付费检查中间件**：创建一个中间件，用于检查用户是否有足够的余额或免费额度
- **下载处理器修改**：在`paper_handler.go`的下载相关函数中添加付费检查

### 4. 实现逻辑
1. **配置读取**：启动时从.env文件读取各服务的收费标准
2. **付费检查**：用户请求下载时，检查对应服务的收费标准
3. **免费逻辑**：如果配置为0，则直接允许下载
4. **付费逻辑**：如果配置大于0，则检查用户余额或使用次数
5. **扣减费用**：付费成功后，扣减用户余额或使用次数

### 5. 支持的服务类型
- 论文下载（PaperDownload）
- 格式检查（FormatCheck）
- 格式修复（FormatFix）
- 报告下载（ReportDownload）
- 比较功能（Compare）

### 6. 灵活性设计
- 支持随时修改.env文件中的配置，无需重新编译
- 支持不同服务独立配置
- 支持免费和付费模式自由切换
- 支持精确到分的收费标准

### 7. 与现有系统集成
- 与现有会员系统兼容
- 与现有支付系统集成
- 支持使用次数和余额两种付费方式

这个方案允许管理员通过简单修改.env文件，灵活配置不同服务的收费模式，实现一次配置，全局生效。
