# 论文格式模板选择与生成设计

## 1. 系统架构概述

当前系统已经具备了基本的论文格式检查和修复功能，主要包括：
- `FormatStandard` 模型：存储论文格式标准，包含ID和JSON格式的Settings
- `Paper` 模型：关联特定的FormatStandard
- `PaperService`：提供论文上传、检查、修复等功能

## 2. 完整流程设计

### 2.1 模板管理流程

**获取可用模板列表**：
```
GET /api/v1/format-standards
```
- 后端返回所有公开的格式标准列表
- 每个模板包含ID、名称、描述、是否默认等信息

### 2.2 前端选择模板流程

1. **获取模板列表**：前端调用API获取所有可用的论文格式模板
2. **展示模板选项**：以卡片形式展示，包含模板名称、所属机构/学校等信息
3. **用户选择**：用户选择适合自己的模板
4. **传递模板ID**：将选中的模板ID与论文一起提交给后端

### 2.3 后端处理流程

**上传论文并指定模板**：
```
POST /api/v1/papers
```
- 请求体包含：论文文件、标题、描述、`format_standard_id`
- 后端验证模板ID是否存在
- 创建论文记录，关联指定的格式标准

**检查论文格式**：
```
POST /api/v1/papers/:id/check
```
- 可以再次指定不同的`format_standard_id`
- 后端使用指定的模板进行格式检查
- 返回检查结果

**生成修正文件**：
- 检查完成后，自动生成修正建议
- 支持批量修复或选择性修复
- 生成修正后的文件路径保存到Paper.DocInfo中

### 2.4 生成文件检查流程

1. **验证文件存在**：检查修正后的文件是否成功生成
2. **文件完整性检查**：验证文件大小、格式、内容是否完整
3. **格式二次验证**：使用相同模板再次检查生成的文件
4. **返回检查结果**：包含生成状态、文件信息、验证结果

## 3. 关键代码优化点

- 为`FormatStandard`模型添加`Institution`字段，标识所属学校/机构
- 支持模板分类，按学校、文档类型等进行分组
- 添加模板预览功能，展示模板的主要设置项

### 3.2 改进检查流程

- 在`CheckPaperFormat`中增强模板处理逻辑
- 确保生成的修正文件路径正确保存
- 添加生成文件的自动验证步骤

### 3.3 前端集成建议

- 实现模板卡片选择界面
- 提供模板详情查看功能
- 显示模板的主要格式要求摘要
- 支持模板搜索和过滤

## 4. API设计建议

### 4.1 获取模板列表
```
GET /api/v1/format-standards
```
- 支持按机构过滤：`?institution=重庆工程学院`
- 支持按文档类型过滤：`?document_type=本科论文`

### 4.2 上传论文指定模板
```
POST /api/v1/papers
```
- 请求体：
  ```json
  {
    "title": "我的论文",
    "description": "描述",
    "format_standard_id": "uuid-of-template",
    "file": "上传的文件"
  }
  ```

### 4.3 检查生成文件
```
GET /api/v1/papers/:id/generated-file-check
```
- 返回生成文件的检查结果
- 包含文件状态、大小、格式验证结果

## 5. 数据模型优化

type FormatStandard struct {

## 6. 实现步骤

1. **前端**：实现模板选择界面，调用API获取和提交模板
2. **后端**：确保模板ID验证和使用逻辑正确
3. **生成文件**：优化修正文件生成流程
4. **验证机制**：添加生成文件的自动验证
5. **反馈机制**：向用户展示生成状态和验证结果

## 7. 预期效果

- 用户可以方便地选择适合自己学校/机构的论文格式模板
- 系统根据选择的模板自动检查和修正论文格式
- 生成的文件经过验证，确保符合要求
- 完整的反馈机制，让用户了解生成状态和结果

这个设计充分利用了现有系统的功能，只需要进行少量的优化和增强，就能满足用户的需求。
